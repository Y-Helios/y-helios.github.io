and ten million tubs = 33333332.000000

mint = 3.333333 and a million mints = 3333333.333333

  1. 通常cout会删除结尾的0，如使3333333.250000显示为3333333.25，调用cout.self()将覆盖这种行为
  2. 结果出现了问题即float和double各自的精度的原因
  3. cout打印6位小数

a = 2.3e+22

b - a = 0

原因：2.34E+22是一个小数点左边有23位的数字，加上1，就是在第23位加1。但float类型只能表示数字中的前六位或前七位，精度不够，修改第23位对值没有任何影响。

表达式中转换

  • 一些类型在出现时就会自动转换；其次有些类型在与其他类型同时出现在表达式中时会被转换

  • 自动转换：C++将bool，char, unsinged char ,singed char 和 short值转换为int ，这些转换被称为整型提升。比如：

    short chickens = 20; 其实是首先将int 类型的20 转换为 short类型再存放再chicken中

  • 不同类型进行算术运算时，也会进行一些转换，他会按照如下的列表进行查询：

    1. 如果有一个是long double ,则另一个也会转换为long double

    2. 如果有一个是double ，则另一个也会转换为double

    3. 如果有一个类型是float，则另一个也会转换为float

    4. 否则，说明操作数都是整型，因此执行整型提升

    5. 如果一个有符号，另一个无符号，则无符号操作数的级别比有符号的高，会把有符号的转换为无符号的

    6. 如果有符号类型可以表示无符号类型的所有可能取值，则将无符号操作数转换为有符号

    7. 否则，将两个操作都转换为有符号类型的无符号版本
